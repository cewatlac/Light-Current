Inter-VLAN routing

Inter-VLAN routing يستخدم لعزل traffic بين أنظمة مثل CCTV وBMS وAccess Control وGuest Wi-Fi مع التحكم في من يسمح له بالوصول لمن.

Inter-VLAN routing separates network traffic between systems such as CCTV, BMS, access control, servers, management, and guest networks while controlling who may reach whom.

Technical Visuals

01

Quick Definition

VLAN ليس اسم فقط؛ هو رقم، subnet، gateway، trunk/access ports، وfirewall rules. أي خطأ في tagging أو gateway قد يعزل الجهاز أو يفتح وصول غير مطلوب.

A VLAN is not only a name. It includes VLAN ID, subnet, gateway, trunk/access ports, and firewall rules. Wrong tagging or routing can isolate devices or expose systems.

02

Why It Matters

العزل الصحيح يقلل broadcast، يسهل troubleshooting، ويمنع خلط security systems مع شبكات عامة أو غير مصرح بها.

Correct segmentation reduces broadcast noise, improves troubleshooting, and prevents security systems from mixing with public or unauthorized networks.

03

Where It Appears

يظهر في network architecture، VLAN matrix، switch port schedule، firewall rules، IP plan، cybersecurity review، وcommissioning isolation test.

You meet it in network architecture, VLAN matrices, switch-port schedules, firewall rules, IP plans, cybersecurity reviews, and isolation tests.

04

Practical Examples

  • Put CCTV cameras, NVR/VMS servers, access-control controllers, BMS controllers, and management interfaces in controlled segments.
  • Define which VLANs may talk to servers, DNS/NTP, head-end software, and vendor remote-support paths.
  • Check trunk allowed VLANs and access-port VLAN assignment before blaming device configuration.
  • Test inter-VLAN rules from a real endpoint, not only from the firewall console.
DESIGN

Design Considerations

في التصميم، وثق VLAN ID، name، subnet، gateway، DHCP/static policy، trunk list، allowed services، وowner لكل rule.

In design, document VLAN ID, name, subnet, gateway, DHCP/static policy, trunk list, allowed services, and owner for each rule.

SITE

Installation Considerations

في الموقع، طابق switch port مع device label، access/trunk mode، allowed VLANs، IP/subnet، وdefault gateway.

On site, match switch port to device label, access/trunk mode, allowed VLANs, IP/subnet, and default gateway.

TEST

Testing And Commissioning

اختبر ping/port access المسموح، منع الوصول غير المسموح، DNS/NTP، server reachability، وlogging للأحداث المهمة.

Test allowed ping/ports, blocked paths, DNS/NTP, server reachability, and logging for important events.

MISTAKES

Common Mistakes

  • Leaving ELV devices on the default VLAN.
  • Allowing all VLANs on every trunk without reason.
  • Testing only connectivity and forgetting denied traffic.
TROUBLESHOOT

Troubleshooting Scenario

لو جهاز لا يصل للسيرفر، راجع port VLAN، trunk allowed list، IP/subnet، gateway، firewall rule، ثم server service نفسه.

If a device cannot reach a server, check port VLAN, trunk allowed list, IP/subnet, gateway, firewall rule, then the server service.
